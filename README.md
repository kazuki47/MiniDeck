# MiniDeck

MiniDeckは、カスタマイズ可能なボタンを配置してアプリケーションの起動、URL表示、キーボードショートカットを実行できるデスクトップアプリケーションです。

## 機能

- **カスタマイズ可能なボタン**: 1〜3行、1〜5列の範囲でボタンを配置可能
- **アプリケーション起動**: ボタンクリックで任意のアプリケーションを起動
- **URLを開く**: 登録したWebページを既定のブラウザーで表示
- **キーボードショートカット**: ボタンクリックでキーの組み合わせを送信
- **マルチアクション**: 1つのボタンで複数のアクションを順番に実行し、待機時間、失敗時の停止・続行、実行前確認を設定可能
- **非アクティブ型メインウィンドウ**: MiniDeckをクリックしても作業中のアプリからキーボードフォーカスを奪わない
- **画像サポート**: ボタンにカスタム画像を設定可能
- **画像の永続保存**: 選択したボタン画像と背景画像を `%LocalAppData%\MiniDeck\Images` にコピーし、再ビルド後も保持
- **画像とアイコンの管理**: 使用場所、未使用画像、見つからない画像を一覧で確認し、安全な削除や再選択が可能
- **画像の重複防止とアイコンキャッシュ**: 同じ内容の画像を再利用し、Windowsシェルアイコンの再抽出を抑制
- **状態を表示するボタン**: アプリの起動状態、マイク、システム音声のミュート状態に応じて表示名・画像・色を切り替え
- **グリッド型ボタン編集**: メイン画面と同じ配置で選択、編集、並べ替え、複製、クリアが可能
- **編集用コピー＆貼り付け**: ボタンとページをMiniDeck内でコピーし、別の位置やページへ貼り付け可能
- **ページ順の編集**: ページを縦一覧へ表示し、ドラッグ＆ドロップで並べ替えて保存可能
- **ドラッグ＆ドロップ登録**: `.exe`、`.lnk`、`.url`、フォルダー、通常ファイル、URL文字列からボタンを作成
- **正方形ボタン**: メイン画面と設定画面で80×80の共通ボタンを使用
- **ページ切り替え**: 複数ページ時はボタングリッド上段の左右に80×80の移動ボタンを表示し、用途別ページを切り替え
- **設定の永続化**: アプリ再起動後も設定が保持される
- **背景カスタマイズ**: 背景色や画像、透明度の調整が可能
- **初期背景**: 新規設定では背景画像を使用せず、不透明な白色背景を使用
- **レイアウト設定の一括適用**: スライダー操作中は保存せず、「適用」または「OK」でまとめて保存
- **一般設定**: Windowsログイン時の自動起動と、メインウィンドウの常に最前面表示を切り替え可能

## 動作環境

- Windows 10以降
- .NET Framework 4.7.2以降
- 最小画面解像度: 1024x768

## ビルド方法

### 前提条件
- Visual Studio 2019以降
- .NET Framework 4.7.2 Developer Pack

### ビルド手順

1. リポジトリをクローン
```bash
git clone <repository-url>
cd MiniDeck
```

2. Visual StudioでMiniDeck.slnを開く

3. ビルド構成を選択（Debug/Release）

4. ソリューションをビルド

### 自動ビルドスクリプト

プロジェクトには複数のビルドスクリプトが用意されています：

- `CreateSimpleDistribution.bat` - シンプルな配布パッケージ作成
- `BuildInstaller.bat` - MSIインストーラー作成
- `CleanBuildAndDist.bat` - クリーンビルドと配布

## 使用方法

1. MiniDeck.exeを実行
2. メインウィンドウ上部の「設定」をクリック
3. 「ボタン設定」タブ上部で編集するページを選択。用途を分ける場合は「追加」でページを作成
4. 編集するボタンを選択し、「編集」をクリック。または、登録したいファイル・フォルダー・URLをボタンへドロップ
5. アクションタイプを選択：
   - **なし**: 何も実行しない
   - **キーボードショートカット**: 指定したキーの組み合わせを送信
   - **アプリ・ファイルを開く**: 指定したアプリ、ファイル、フォルダー、ショートカットをWindowsの関連付けで開く
   - **URLを開く**: `http://` または `https://` のURLを既定のブラウザーで表示
   - **マルチアクション**: 複数のショートカット、アプリ・ファイル、URLを登録順に実行
6. マルチアクションの場合は「アクションを編集...」から内容と実行順、待機時間を設定する
7. キーボードショートカットの場合は「キーを入力」をクリックし、登録するキーの組み合わせを押す
8. URLの場合はURLを入力し、必要に応じて「テスト実行」で確認する
9. 必要に応じてボタンの表示名、画像、状態表示を設定し、ボタン編集画面の「OK」で編集用コピーへ反映
10. 設定画面の「適用」または「OK」で全ページの変更を反映・保存

MiniDeckのメインウィンドウは、ボタンをクリックしてもアクティブになりません。そのため、キー操作はMiniDeckではなく、直前から使用しているアプリケーションへ送信されます。設定画面は通常のウィンドウとして開くため、文字入力やショートカット登録が可能です。

ページが2枚以上になると、メイン画面のボタングリッド上段左側に「前のページ」、右側に「次のページ」の80×80ボタンが表示されます。先頭・末尾では移動できない側が無効になり、1ページだけの場合は両方とも表示されません。最後に表示していたページは自動保存され、次回起動時に復元されます。

### ボタン設定

- 設定画面には、メイン画面と同じ行列・順序で正方形ボタンが表示されます。
- 上部のページ一覧から編集対象を切り替えられます。「追加」「複製」「名前変更」「削除」でページを管理できます。
- 「ページを並べ替え...」を押すと、現在のページが縦一覧で表示されます。ページをドラッグ＆ドロップし、並べ替え画面の「OK」で編集内容へ反映します。
- 並べ替えた順番は、設定画面の「適用」または「OK」で保存されます。並べ替え画面または設定画面で「キャンセル」した場合は破棄されます。
- ページの「コピー」「貼り付け」は、コピーした内容を独立した新規ページとして追加します。
- ページ削除時は設定済みボタン数を表示して確認し、最後の1ページは削除できません。
- ボタンのクリックで選択し、ダブルクリックまたは「編集」で内容を変更できます。
- ボタン同士をドラッグすると2つの配置を入れ替えられます。「前へ」「次へ」では選択ボタンを1つずつ移動できます。
- 「複製」は選択した設定を次の空きボタンへコピーし、「クリア」は選択位置を未設定へ戻します。
- ボタンの「コピー」「貼り付け」で別の位置やページへ設定を複製できます。設定済みボタンへの貼り付け時は上書き確認が表示されます。
- 空きボタンには「＋」が表示されます。
- コピー内容はOSのクリップボードではなくMiniDeck内に保持され、同一起動中に設定画面を開き直しても利用できます。ボタンとページのうち、最後にコピーした種類が保持されます。
- 編集中のボタンとページの変更はコピーに保持されます。「適用」または「OK」で全ページを一括保存し、「キャンセル」で未適用の変更を破棄します。
- 新しく選択したボタン画像は、ビルド出力ではなくユーザーのMiniDeck画像フォルダーへ保存されます。
- デスクトップやエクスプローラーから`.exe`、`.lnk`、`.url`、フォルダー、通常ファイルを直接ボタンへドロップできます。
- ブラウザーなどから`http://`または`https://`のURL文字列をドロップできます。
- 表示名とWindowsシェルのアイコンは自動設定されます。設定済みボタンへのドロップは上書き確認が表示されます。
- 複数項目の同時ドロップには対応していません。1ボタンにつき1項目をドロップしてください。

### 状態表示

- ボタン編集画面の「状態表示」で「アプリの起動状態」「マイクのミュート状態」「システム音声のミュート状態」を選択できます。
- アプリ状態は「アプリ・ファイルを開く」アクションに設定した`.exe`、または`.exe`への`.lnk`を監視します。
- マイクとシステム音声は、Windowsの既定入力デバイスと既定出力デバイスのミュート状態を使用します。
- 状態ON時の表示名・画像・背景色と、状態OFF時の背景色を設定できます。状態ONの表示名や画像が空の場合は通常設定を使用します。
- 状態は1秒ごとに更新されます。ボタンへマウスを合わせると、現在の判定結果をツールチップで確認できます。
- 音声デバイスがない、権限の異なるアプリを確認できないなどの理由で状態取得に失敗した場合は、通常の表示名・画像と透明背景へ戻ります。ボタンのアクションはそのまま実行できます。
- 状態変化は実行時表示だけに反映され、状態更新のたびに設定ファイルを保存することはありません。

### マルチアクション

- ボタン編集画面のアクション種別で「マルチアクション」を選択し、「アクションを編集...」を開きます。
- キーボードショートカット、アプリ・ファイル、URLを最大50個まで追加できます。
- 左側の一覧が実行順です。「↑ 上へ」「↓ 下へ」で順番を変更し、「追加」「削除」で内容を管理します。
- 各アクションに、実行後から次のアクションまでの待機時間を0～600000ミリ秒で設定できます。
- 失敗時は「その時点で停止」または「残りのアクションを続行」を選べます。
- 誤操作を避けたいボタンでは、実行前確認を有効にできます。
- 実行中は青、成功時は緑、失敗時は赤の背景になり、表示名とツールチップでも状態を確認できます。成功表示は自動的に通常表示へ戻り、失敗表示は次の実行まで残ります。
- 実行状態の変化だけでは設定ファイルを保存しません。同じマルチアクションを実行中に重ねて起動することもありません。

### レイアウト設定

- 行数、列数、背景透明度、ボタン透明度は、スライダーを動かしただけでは保存されません。
- 「適用」をクリックすると設定画面を開いたまま保存し、「OK」をクリックすると保存して閉じます。
- 「キャンセル」をクリックすると、未適用のスライダー・ボタン・ページ変更を破棄します。
- ボタン数を減らすと全ページで末尾のボタン設定が削除されるため、適用前に確認メッセージが表示されます。
- 適用後は、行数と列数に合わせてメインウィンドウのサイズが調整されます。

背景画像もユーザーのMiniDeck画像フォルダーへ保存されます。以前のバージョンでビルド出力内に保存した画像が見つからない場合は、一度画像を選択し直してください。

### 画像管理

- 設定画面の「画像管理」タブから「画像管理を開く...」を選びます。
- 一覧にはボタン画像、背景画像、自動取得したWindowsシェルアイコンの状態と使用場所が表示されます。
- 「見つからない」画像を選び「画像を選択し直す...」を押すと、そのパスを使用しているボタンまたは背景をまとめて修復できます。
- 再選択した内容は画像管理画面の「OK」で設定画面の下書きへ反映され、設定画面の「適用」または「OK」で保存されます。
- 「未使用」のMiniDeck管理画像だけを個別または一括で削除できます。使用中の画像、および設定画面を開いた後にまだ適用していない旧設定の画像は削除されません。
- 未使用画像の削除は直ちに行われ、設定画面の「キャンセル」では元に戻りません。

### 一般設定

- 「起動時に自動起動」を有効にすると、現在のWindowsユーザーのログイン時にMiniDeckが起動します。
- 「常に最前面」を無効にすると、MiniDeckを他のウィンドウより背面へ移動できるようになります。
- 変更は「適用」または「OK」で反映・保存され、「キャンセル」では未適用の変更が破棄されます。

### プロジェクト構造

```
MiniDeck/
├── MiniDeck/                 # メインプロジェクト
│   ├── Models/              # データモデル
│   ├── ViewModels/          # MVVMビューモデル
│   ├── Services/            # ビジネスロジック
│   ├── Controls/            # カスタムコントロール
│   └── Resources/           # リソースファイル
├── MiniDeckSetup/           # インストーラープロジェクト
├── packages/                # NuGetパッケージ
└── *.bat                    # ビルドスクリプト
```

### 主要なクラス

- **MainWindow**: メインウィンドウ
- **MainViewModel**: メインビューのロジック
- **ActionButton**: ボタンの動作を定義するモデル
- **ButtonGridControl**: メイン画面と設定画面で共有するボタングリッド
- **ButtonDropService**: ドロップデータの検証とボタン設定への変換
- **ButtonStateMonitorService**: アプリ起動状態とWindows音声デバイスのミュート状態を監視
- **MacroActionStep**: マルチアクション内の1手順と待機時間を保持
- **EditorClipboardService**: ボタンとページの編集用コピーを独立したデータとして保持
- **ShellIconService**: Windowsシェルアイコンの取得
- **ImageStorageService**: 画像の永続保存、内容重複の防止、シェルアイコンキャッシュ
- **ImageAssetService**: 画像の使用状況・欠損判定と安全な未使用画像削除
- **SettingsService**: 設定の保存・読み込み
- **ActionService**: 単一アクションとマルチアクションの順次実行、待機、失敗制御

### ページ対応データ基盤

- 設定形式バージョン4では、ページ単位のボタン設定に状態表示とマルチアクション設定を保存します。
- 従来の `Buttons` は初回読み込み時に「メイン」ページへ自動移行されます。
- 旧形式からの移行前には `settings.pre-v4.xml`、通常保存前には `settings.xml.bak` が作成されます。
- 設定画面からページの追加、複製、名前変更、切り替え、削除ができます。
- ページとボタンは「適用」または「OK」でまとめて保存され、「キャンセル」では破棄されます。
- メイン画面には通常ボタンと同じデザインの前後移動ボタンがあり、最後に開いたページを復元します。
